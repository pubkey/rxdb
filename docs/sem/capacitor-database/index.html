Online is Optional

RxDB adopts an offline-first approach, keeping your app fully functional even without a connection. Data is stored locally on the Capacitor client and seamlessly replicated in the background, and you can even skip the backend entirely if you choose.

  • Keep your Capacitor app running offline
  • Run local queries with zero latency
  • Simplify and speed up development
  • Reduces backend load and scales better

Realtime Applications made easy

In RxDB, everything is observable. Query Results, Documents, Document Fields, Events.

  • Whenever data changes, the UI updates.
  • Realtime events across components, browser tabs and user devices
  • Supports Capacitor and all major frameworks:
