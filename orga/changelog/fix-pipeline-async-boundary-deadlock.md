- FIX `RxPipeline` deadlock when the handler reads from or writes to the destination collection from an async callback that breaks the synchronous stack trace (such as a `setTimeout`, `setImmediate` or subscriber callback), because the stack-based detection in `waitBeforeWriteFn` could no longer see the pipeline's flagged function and would wait for the pipeline to become idle while still inside the pipeline itself. A module-level counter of currently running handlers is now used in addition to the stack check so the handler's execution context is detected across async boundaries.
