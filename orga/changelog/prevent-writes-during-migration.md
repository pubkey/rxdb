- FIX writes to an `RxCollection` racing with a running schema migration could surface as confusing `RC_PUSH` errors. Writes (`insert`, `bulkInsert`, `upsert`, `bulkUpsert`, `incrementalUpsert`, `bulkRemove`, `RxDocument` modifications, attachment writes) are now blocked with a clear `COL25` error while a migration is pending or running. The new `isWriteAllowed()` helper consolidates the "closed" and "migrating" checks at every write call site, and the migration plugin now sets `migrationInProgress` eagerly on collection creation so writes are also refused before `startMigration()`/`migratePromise()` is called.
